## 🎯 БРИФ ДЛЯ ИИ-АГЕНТА: Реализация DRAKON в Mermaid

### Контекст
Нужно создать внешний плагин для Mermaid.js, который поддерживает диаграммы типа `drakon` с синтаксисом, описанным ниже. Плагин должен рендерить SVG строго по правилам ДРАКОН Паронджанова.

---

## 📜 ЭТАП 1: ГРАММАТИКА (файл `drakon.langium`)

Агенту нужно создать грамматику, которая парсит текст вида:

```
drakon
  Начало: "Запуск процесса"
  Вопрос: "Есть данные?" ->
    Да: "Обработать" ->
      "Сохранить"
    Нет: "Ожидать" ->
      "Конец"
  Конец: "Завершение"
```

**Требования к грамматике:**
- Корневой элемент — `DrakonDiagram`
- Блоки: `Начало`, `Конец`, `Вопрос`, `Действие`, `Ввод/Вывод` (можно использовать ключевые слова)
- У каждого блока — уникальный `id` и `text`
- У `Вопроса` — два выхода: `Да` и `Нет`, каждый ведёт к блоку
- У остальных — один выход (к следующему блоку)

## 🗃️ ЭТАП 2: ПАРСЕР И БАЗА ДАННЫХ

**Файл `drakonDb.ts`** должен реализовывать:

```typescript
interface DrakonNode {
  id: string;
  type: 'start' | 'end' | 'question' | 'action' | 'io';
  text: string;
  x?: number;        // вычисляется на этапе layout
  y?: number;
  width?: number;
  height?: number;
}

interface DrakonEdge {
  from: string;
  to: string;
  label?: 'Да' | 'Нет';  // только для вопросов
}
```

**Функции:**
- `addNode(node: DrakonNode)`
- `addEdge(edge: DrakonEdge)`
- `getNodes(): DrakonNode[]`
- `getEdges(): DrakonEdge[]`
- `getStartNode(): DrakonNode`
- `validate()`: проверить, что:
  - ровно один `start` и ровно один `end`
  - у всех `question` есть два выхода (с `Да` и `Нет`)
  - нет циклов (опционально, но желательно)

## 🧩 ЭТАП 3: АЛГОРИТМ КОМПОНОВКИ (САМОЕ ВАЖНОЕ!)

**Файл `layoutEngine.ts`** — реализовать алгоритм, который расставляет координаты для всех нод строго по правилам ДРАКОН:

### Правила для реализации:

1. **Вертикальная магистраль** — основная линия идёт сверху вниз.
2. **Все ноды размещаются на этой вертикали** (координата X у всех одинаковая).
3. **Ветвление (у вопроса)** — правая ветка (`Да`) и левая ветка (`Нет`) **отходят горизонтально** под прямым углом, а затем **возвращаются на магистраль** ниже.
4. **Запрещены пересечения линий** — нужно использовать дополнительные вертикальные "шины" для возврата веток.
5. **Запрещены наклонные и изогнутые линии** — только `M x y L x y` (прямые углы).
6. **Иконки (блоки) крепятся только к вертикальным линиям**.

### Алгоритм (упрощённо):

```typescript
function layout(nodes: DrakonNode[], edges: DrakonEdge[]): LayoutResult {
  // 1. Найти start-ноду — она наверху (y=0)
  // 2. Обойти граф в глубину (DFS) начиная со start
  // 3. Для каждой ноды:
  //    - Если это action/io/end → следующая нода ставится ниже с отступом
  //    - Если это question:
  //        * Ветка "Да" уходит вправо
  //        * Ветка "Нет" уходит влево
  //        * Обе ветки возвращаются на магистраль ниже
  //        * Все блоки внутри ветки выравниваются по своей вертикали
  // 4. Если есть вложенные вопросы → рекурсивно обработать
  // 5. После расстановки всех координат — построить пути (пути идут отдельно)
}
```

**Важно:** агент должен реализовать **простой алгоритм без пересечений**, даже если это означает добавление дополнительных "пустых" вертикальных линий-заглушек.

## 🎨 ЭТАП 4: РЕНДЕРИНГ (SVG)

**Файл `svgBuilder.ts`** — генерирует SVG на основе размеченных нод:

1. **Блоки**:
   - `start` / `end`: скруглённые прямоугольники (эллипсы)
   - `question`: ромб (обычно, но в ДРАКОН это может быть прямоугольник с наклонными сторонами — уточните по ГОСТу)
   - `action`: прямоугольник
   - `io`: параллелограмм

2. **Линии**:
   - Все линии строятся как `path` с атрибутом `d="M x1 y1 L x2 y2 L x3 y3 ..."`
   - Толщина линии: 2px
   - Цвет: чёрный (#000)

3. **Стрелки**:
   - На конце каждой линии — стрелка (треугольник)
   - Реализовать через `marker-end`

4. **Текст**:
   - Внутри блоков — центрированный текст
   - Шрифт: sans-serif, размер 14px

## 🔌 ЭТАП 5: РЕГИСТРАЦИЯ ПЛАГИНА

**Файл `index.ts`**:

```typescript
import { registerExternalDiagrams } from 'mermaid';

const drakonPlugin = {
  id: 'drakon',
  detector: (text: string) => /^\s*drakon\b/.test(text),
  loader: async () => {
    const { parse, render } = await import('./parser/drakonParser');
    return { parse, render };
  }
};

registerExternalDiagrams([drakonPlugin]);
```
